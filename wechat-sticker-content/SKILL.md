# 微信贴图号内容生成

## Core Principle

把贴图号内容当成“可转发的认知贴纸”，不是文章、课程、公众号长图或小红书轮播。

贴图号默认优先做 **单张 3:4 竖版图**。它应该像一张能被截图、收藏、转发的“观点贴纸”，而不是一组需要滑动阅读的内容。多图只在每张都能独立成为贴纸时使用。

首要判断不是“知识是否完整”，而是：

> 用户转发这张图，是想让别人怎么看自己？

优先服务四类转发动机：

| 转发动机 | 用户潜台词 | 适合内容 |
| --- | --- | --- |
| 身份表达 | 我关注前沿、我懂AI | AI认知判断、趋势观点 |
| 认知优越感 | 我不是跟风，我搞懂了 | 概念翻译、框架图、对比图 |
| 帮助别人 | 这份资料对你有用 | 工具选择、行动清单、模板 |
| 焦虑缓解 | 原来不是我一个人这样 | 职场焦虑解释、学习误区、安抚型判断 |

## Mandatory Workflow

每次用户给出待发布内容时，按以下顺序输出：

1. **社交价值判断**：指出这段内容主要满足哪一种或哪几种转发动机。
2. **发布结论**：明确说“适合发布贴图号”或“这个内容不适合发布贴图号”。
3. **不适合时直接停止**：如果不值得发布，只给出简短原因和一个改造方向；不要继续生成标题和生图内容。
4. **适合时生成发布方案**：输出标题、推荐图片数量、是否分多次发布、每张图的内容形式和可直接生图的文案。
5. **先做单图判断**：如果素材可以被压缩成一张清单、榜单、对比图、四象限或观点海报，优先推荐1张图；不要因为素材自带编号就自动拆成多图。

## Publishability Formula

用这个公式判断：

> 可发布性 = 转发身份 + 单图成立 + 3秒理解 + 情绪/认知钩子 + 可视化结构

逐项打分，满分10分：

- **转发身份 0-2分**：用户转发后是否显得更清醒、更专业、更前沿、更会提效？
- **单图成立 0-2分**：单张图被截取或转发时，是否还能独立表达完整意思？
- **3秒理解 0-2分**：标题和结构是否能在微信缩略图/聊天流中迅速看懂？
- **情绪/认知钩子 0-2分**：是否有反常识、共鸣、焦虑解释、态度判断或明确收益？
- **可视化结构 0-2分**：是否能做成对比、三段式、清单、流程或一张轻量卡片？

判断标准：

- **8-10分**：强烈适合发布，优先做成贴图号内容。
- **6-7分**：可以发布，但必须改成更强标题或更轻结构。
- **0-5分**：不适合发布贴图号。直接说明不适合，不生成完整发布方案。

## Content Forms

优先选择这些形式：

| 优先级 | 形式 | 使用场景 | 图片数量 |
| --- | --- | --- | --- |
| S | 单句观点卡 | 强态度、强判断、身份表达 | 1张 |
| S | 讽刺清单/榜单海报 | 四件套、三件事、工具清单、消费清单、身份标签 | 1张 |
| S | 三段式认知卡 | 新手/进阶/高手，过去/现在/未来 | 1张优先 |
| A | 对比卡 | 错误/正确，搜索思维/任务思维 | 1张优先 |
| A | 行动清单卡 | 步骤、模板、工具选择 | 1张优先 |
| B | 系列图解 | 一个主题拆成多张独立卡 | 3张，谨慎使用 |
| C | 信息长图 | 复杂体系、工具全景 | 少用，不作为主力 |

多图原则：

- 单图优先。贴图号不是轮播号，默认不要把内容拆成多图。
- 多图是分发方式，不是阅读方式。
- 每张图都要能单独转发、单独成立。
- 不要把一篇文章硬拆成6-9张依赖顺序阅读的图。
- 不要因为素材里有01/02/03/04，就自动拆成4张；编号清单通常更适合做成一张榜单/清单海报。
- 只有当每个编号都是完全独立的观点、且拆开后转发欲不下降时，才考虑多图。
- 多图最多建议3张；5张以上通常更像小红书轮播，除非用户明确要求。
- 如果拆图会削弱原本的整体梗、榜单感、讽刺感或身份表达，就必须做单图。

## Ratio Rules

贴图号画面比例默认遵守：

- **默认比例：3:4**。适合竖向认知海报、清单、榜单、对比图，标题和正文都有足够空间。
- **谨慎使用1:1**。只适合极短单句、表情包式观点、强符号贴纸。
- **不推荐长图**。长图更像公众号或知识库截图，不利于聊天流快速转发。
- 输出画面建议时，如果没有特殊原因，比例写3:4。

## Rewrite Rules

将普通AI知识改成贴图号表达时，遵守：

- 观点先行，知识补位，转发优先。
- 一张图只讲一个观点。
- 一个整体梗、一张榜单、一个“四件套/三件事/避坑清单”优先合成一张图。
- 标题12-20字优先，尽量像一句可转发判断。
- 正文控制在30-80字；清单控制在3-5点。
- 少做“什么是X”的平铺标题，多做“别再把X理解成Y了”。
- 少做完整知识课，多做认知切片。
- 少做长图体系，多做单张成立。
- 看到编号结构时，先判断它是“系列”还是“榜单”。如果标题天然是“四件套/清单/排行榜/对照表”，优先按单张榜单图处理。

常用标题改造：

| 普通标题 | 贴图号标题 |
| --- | --- |
| 什么是AI Agent | 别再把Agent理解成聊天机器人了 |
| 普通人如何学习AI | 普通人学AI，最不该先学工具 |
| AI会不会取代人 | AI不会淘汰你，但会暴露你不会拆任务 |
| ChatGPT和Claude怎么选 | 新手别乱买AI会员，先看这张表 |
| 如何写Prompt | 问AI前，先把这3件事说清楚 |

## Output Template

对用户素材输出以下结构。

### 不适合发布时

```markdown
**社交价值判断**
这段内容的社交价值偏弱，主要问题是：{原因}。

**发布结论**
这个内容不适合发布贴图号。

**改造方向**
可以把它改成：{一个更适合贴图号的角度}。
```

### 适合发布时

```markdown
**社交价值判断**
- 主要转发动机：{身份表达/认知优越感/帮助别人/焦虑缓解}
- 可发布性评分：{x}/10
- 为什么值得发：{一句话说明用户为什么会转发}

**发布方案**
- 推荐标题：{标题}
- 推荐图片数：{默认1张；仅在必要时3张}
- 是否分多次发布：{不需要/建议分为x次}
- 内容形式：{单句观点卡/讽刺清单/榜单海报/三段式认知卡/对比卡/行动清单卡/系列图解}

**生图内容**

第1张｜{形式}
标题：{图上大标题}
副标题：{可选}
正文：
{30-80字或3-5个要点}
底部总结：{一句认知总结}

第2张｜{形式}
...

**画面建议**
- 比例：{默认3:4；只有极短单句才建议1:1}
- 视觉：{简洁知识卡/轻贴纸感/统一AI角色/对比结构/清单结构/榜单海报}
- 注意：标题大、正文少、每张单独成立。
```

## Quality Bar

生成前做最后检查：

- 用户转发这张图是否能显得更懂AI？
- 这张图是否不看配文也能成立？
- 微信聊天里缩略图是否能看清标题？
- 是否优先尝试了3:4单图，而不是惯性拆成多图？
- 如果素材有编号，拆开后是否会削弱整体梗、榜单感或讽刺感？
- 是否避免了长文章、纯新闻、纯参数、空泛焦虑？
- 是否已经把知识翻译成身份表达、判断标准、行动清单或认知对比？

如果任何一个关键检查明显不通过，降低评分；低于6分时不要生成完整发布方案。
